整数溢出

题目:number

先查看保护:

几乎无

主要代码:


和

讲一下程序流程

主要main函数循环：每次使v4数组对应的元素等于readll函数的返回值。
readll：for循环最多可以输入20个字符，每次可以读取一个字符后调用atoll函数把nptr数组上的值返回给V4数组。
(atoll函数解析:https://cplusplus.com/reference/cstdlib/atoll/)

解题思路:

第一种方法:

利用整数溢出覆盖返回地址，然后利用puts函数把函数的got表打印出来从而获取到libc基址然后能找到system函数和”/bin/sh”字符的地址，然后再次重新返回main函数，再次利用整数溢出覆盖返回地址:先把参数pop为”/bin/sh”然后再调用system函数，成功获得shell。

EXP

第二种方法：

需要注意的点

1. ida中可以看出readll函数中的read函数每次只能读取一个字符.


然后gdb中输入的时候也确实只读取了一个字符在栈上，如果对于C语言比较生疏的，或者没有特别仔细地经过gdb调试的，那么就会误以为每次只能发送一个字符，然而事实是你每次可以发送最多20个字符，它会先逐个读取到nptr数组对应的元素内储存，然后利用atoll函数直接赋值给main函数的V4数组。

演示：

利用atoll函数逐个赋值给main函数的V4数组的对应过程

：可以看到在调用readll函数的时候，我输入了123也就是十六进制的0x7b
然后看到rax的值也是0x7b，之后从后面的汇编代码我们可以看到，rax把0x7b赋值给了rdx寄存器。
在看汇编代码: javascript mov qword ptr [rbp + rax*8 - 0x60], rdx可以看到在再次调用readll函数之前，rdx已经赋值给了另外一个地址，这个地址也就是v4的地址了。

跳转到 javascript mov qword ptr [rbp + rax*8 - 0x60], rdx

可以看到rbp + rax*8 - 0x60的值就是0x7fffffffddc0

然后步过就可以看到0x7b的值赋值到了0x7fffffffddc0上，然后下一次循环也就是下一次调用radll输入的数字会存储到0x7fffffffddc8去。

可以看到0x7fffffffddc0与rbp的距离正好是0x60
